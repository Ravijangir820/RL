SLIDE 2: PROBLEM & MOTIVATION

- Solution: Hierarchical RL with Options framework

SLIDE 3: WHAT IS HIERARCHICAL RL?
Key Concepts:
- Standard RL: Agent learns one level of actions (primitive)
- Hierarchical RL: Multiple levels of decision making
- Options Framework: Extend MDP with discrete, reusable sub-skills
- Option Definition: (π, β) = (intra-option policy, termination function)
- Benefit: Temporal abstraction enables better credit assignment
- Real-world: Navigate to kitchen (option) vs move each muscle (primitive action)

SLIDE 4: FLAT Q-LEARNING (BASELINE)
Algorithm:
- Direct mapping: State → Primitive Action (6 actions in this project)
- Tabular Q-table: Q(state, action) - stores expected reward for each state-action pair
- Update rule: Q(s,a) ← Q(s,a) + α[r + γ*max_a' Q(s',a') - Q(s,a)]
- Exploration: ε-greedy (explore random with probability ε)
- Hyperparameters: α=0.1, γ=0.99, ε=0.1
- Training: 20,000 episodes on Taxi-v3 environment
- Pro: Simple, interpretable, proven to work
- Con: Struggles with long sequences, slow credit assignment

SLIDE 5: OPTION-CRITIC (HIERARCHICAL RL)
Algorithm:
- Combines options framework with actor-critic learning
- Three components learned simultaneously:
  1. High-level policy π(o|s): Which option to execute in state s?
  2. Low-level policy π(a|s,o): Which action within option o?
  3. Termination function β(s,o): When to switch to new option?
- Learn Q-values at two levels: Q_O for options, Q_U for actions
- 4 options learned: Different navigation sub-skills
- Same hyperparameters as flat for fair comparison (α=0.1, γ=0.99, ε=0.1)
- Termination probability: β=0.2 (probabilistic option switching)
- Benefit: Temporal abstraction, better credit assignment, more interpretable

- 5×5 configurable grid (supports 8×8, 10×10, larger)
- 4 fixed locations: Taxi start, Passenger, Destination, Depot

- State: (taxi_row, taxi_col, passenger_location, destination)
- Rewards: +20 success, -1 per step, -10 illegal actions
- Terminal condition: Passenger delivered to destination
- Max steps: 200 per episode

Training Configuration:
- Episodes: 20,000 per agent

4. Illegal Actions: Rule violations per episode (lower = better understanding)
5. Efficiency: (Optimal steps / Actual steps) ratio, 1.0 = perfect (higher = better)
6. Illegal Action Ratio: Proportion of illegal moves (lower = better)
