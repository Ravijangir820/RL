SLIDE 3: FLAT Q-LEARNING (BASELINE APPROACH)
Key Points:
- Direct mapping: State → Primitive Action
- Tabular representation: Q(state, action)
- Learning rule: Q(s,a) ← Q(s,a) + α[r + γ*max_a' Q(s',a') - Q(s,a)]
- ε-greedy exploration: explore random actions with probability ε
- Simple but struggles with long sequences due to credit assignment
- Hyperparameters:
  * Learning rate α = 0.1
  * Discount factor γ = 0.99
  * Exploration ε = 0.1

SLIDE 4: HIERARCHICAL RL - OPTIONS FRAMEWORK
Key Concepts:
- Temporal Abstraction: group primitive actions into reusable sub-skills
- Option Definition: (π, β, I) = (intra-option policy, termination function, initiation set)
- Three levels of decision making:
  1. High-level: Which option to execute? Policy π(o|s)
  2. Low-level: Which action within option? Policy π(a|s,o)
  3. Termination: When to switch options? Probability β(s,o)
- Option-Critic Algorithm: learns all three components simultaneously
- 4 options in this project (navigation sub-skills)

SLIDE 5: OPTION-CRITIC IMPLEMENTATION
Architecture:
- High-level Q-values: Q_O(s, option) for option selection
- Low-level Q-values: Q_U(s, option, action) for intra-option actions
- Termination function: β(s,o) ∈ [0,1] probability per state-option pair
- Learning: TD updates using termination-aware bootstrapping
- Hyperparameters (same as flat for fair comparison):
  * Learning rate α = 0.1
  * Discount factor γ = 0.99
  * Exploration ε = 0.1
  * Termination probability β = 0.2 (scheduled termination)
  * 4 options for sub-skill discovery
